MANAGEMENT EVENTS
Management events provide information about management operations
that are performed on resources in your AWS account. These are also
known as control plane operations.
• Configuring security (for example, AWS Identity and Access
Management AttachRolePolicy API operations).
• Registering devices (for example, Amazon EC2 CreateDefaultVpc API
operations).
• Configuring rules for routing data (for example, Amazon
EC2CreateSubnet API operations).
• Setting up logging (for example, AWS CloudTrail CreateTrailAPI
operations).

DATA EVENTS
Data events provide information about the resource operations performed
on or in a resource. These are also known as data plane operations. Data
events are often high-volume activities.
§ Amazon S3 object-level API activity on buckets and objects in buckets.
(for example GetObject, DeleteObject, and PutObject API operations)
§ AWS Lambda function execution activity (the Invoke API).
§ CloudTrail PutAuditEvents activity on a CloudTrail Lake channel that is
used to log events from outside AWS.
§ Amazon SNS Publish and PublishBatch API operations on topics.

INSIGHT EVENTS
CloudTrail Insights events capture unusual API call rate or error rate activity
in your AWS account by analyzing CloudTrail management activity.
§ Your account typically logs no more than 20 Amazon S3 deleteBucket API
calls per minute, but your account starts to log an average of
100 deleteBucket API calls per minute. An Insights event is logged at the
start of the unusual activity, and another Insights event is logged to
mark the end of the unusual activity.
§ Your account typically logs 20 calls per minute to the Amazon
EC2AuthorizeSecurityGroupIngress API, but your account starts to log
zero calls to AuthorizeSecurityGroupIngress. An Insights event is logged
at the start of the unusual activity, and ten minutes later, when the
unusual activity ends, another Insights event is logged to mark the end
of the unusual activity.

SECURING THE LOGS
If CloudTrail logs contains metadata, not application data, why bother
securing them?
Logs may contain PII data such as usernames and team/group
membership. Also can contain detailed configuration information

SECURING THE LOGS
§ Use IAM policies and S3 bucket policies to restrict access to S3 bucket
containing the logs
§ Use SSE-S3 or SSE-KMS to encrypt the logs
§ Configure SNS notifications and log file validation
§ Restrict S3 delete access

ACTING UPON LOGS
§ CloudTrail itself does nothing with logs
§ You can review logs in the CloudTrail console
§ Utilizes CloudWatch Events, to define workflows
§ Detect unusual activity by enabling CloudTrail insights

SNS (SIMPLE NOTIFICATIONS SERVICE)
SNS is a managed service that provides message delivery from publishers
to subscribers (also known as producers and consumers)
§ Publishers communicate asynchronously with subscribers by sending
messages to a topic, which is a logical access point and communication
channel.
§ Clients can subscribe to the SNS topic and receive published messages
using a supported endpoint type, such as Amazon Data Firehose, Amazon
SQS, AWS Lambda, HTTP, email, mobile push notifications, and mobile
text messages (SMS).
