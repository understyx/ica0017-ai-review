Lecture topic: Introduction to Cloud Technologies (ICA0017) and Course Overview

Core concepts:
- Course structure: 3 ECTS, 8 lectures, lab exercises, and a multiple-choice exam (either online via Proctorio or in a physical classroom).
- Goal: Preparation for the AWS Certified Cloud Practitioner exam (AWS as the primary platform, though other providers exist).
- Cloud economics: Transition from CapEx (Capital Expenditure) to OpEx (Operational Expenditure).
- Core advantages of cloud: No upfront hardware costs, flexibility, economies of scale, and speed to market.
- Cloud deployment models: On-premise vs. Public Cloud vs. Hybrid Cloud.
- Service Models (XaaS): Infrastructure as a Service (IaaS), Platform as a Service (PaaS), and Software as a Service (SaaS).
- Scalability vs. Elasticity: Differences between scaling up/out and dynamic provisioning.
- Availability vs. Durability: System uptime versus data preservation.

Key definitions:
- IaaS (Infrastructure as a Service): The provider supplies basic computing infrastructure (servers, storage, networking). User manages the OS and software.
- PaaS (Platform as a Service): Provider manages the underlying infrastructure and OS. User focuses on deploying applications and data.
- SaaS (Software as a Service): Fully functional application delivered over the web. Provider manages everything.
- CapEx: Capital expenses, e.g., buying physical servers upfront.
- OpEx: Operational expenses, e.g., paying for cloud resources on a pay-as-you-go basis.

Important examples:
- IaaS: AWS EC2.
- PaaS: AWS Elastic Beanstalk.
- SaaS: Dropbox, Gmail, Microsoft 365.
- Rapid scalability example: Scaling infrastructure instantly if an application goes viral (e.g., on TikTok).

Likely exam points:
- Distinguishing between IaaS, PaaS, and SaaS (with examples).
- Understanding the shift from CapEx to OpEx in cloud computing.
- The differences between scalability and elasticity.
- The differences between availability and durability.

Questions to review:
1. What are the three primary cloud service models and how do they differ in terms of user management?
2. How does the pay-as-you-go cloud model affect a company's financial planning (CapEx vs. OpEx)?
3. What is the fundamental difference between system availability and data durability?

5-sentence plain-English recap:
This first lecture introduces the "Introduction to Cloud Technologies" course, outlining its structure, requirements, and goal of preparing students for the AWS Certified Cloud Practitioner exam. It covers fundamental cloud concepts, highlighting the shift from expensive upfront hardware investments (CapEx) to flexible, pay-as-you-go models (OpEx). The lecture details the primary cloud service models: IaaS, PaaS, and SaaS, explaining what the provider manages versus the user. Core cloud benefits such as rapid scalability, global reach, and economies of scale are emphasized. Finally, key technical distinctions are made, such as the difference between scaling resources (scalability) and dynamic adjustment (elasticity), as well as system uptime (availability) versus data safety (durability).
