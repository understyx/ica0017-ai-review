# 3. Key concepts and definitions
- **Cloud Computing**: The on-demand delivery of compute power, database storage, applications, and other IT resources through a cloud services platform via the Internet with pay-as-you-go pricing.
- **IaaS (Infrastructure as a Service)**: The provider manages the datacenters and virtualization platform. The user is responsible for installing and maintaining servers (OS, middleware, apps).
- **PaaS (Platform as a Service)**: The provider manages the underlying infrastructure, operating systems, security, patching, and maintenance. The user focuses solely on deploying applications and data.
- **SaaS (Software as a Service)**: A fully functional application delivered over the web. The provider manages everything (e.g., servers, networks, storage, maintenance).
- **CapEx (Capital Expenditure)**: Upfront expenses, such as buying physical servers and data center infrastructure before knowing how much they will be used.
- **OpEx (Operational Expenditure)**: Variable expenses, such as paying for cloud resources on a pay-as-you-go basis only for what is consumed.
- **Virtualization**: Software that manipulates hardware and provides a layer to present a consistent interface, serving as a foundational enabler for cloud computing.
- **Cloud deployment models**:
  - *Public cloud*: Services offered over the public internet (e.g., AWS, Azure, GCP).
  - *Hybrid cloud*: A mixture of public and private clouds.
  - *Private cloud*: On-premises infrastructure (own datacenter).

# 4. Main arguments or theories explained simply
- **Cloud as a utility**: Just like electric and telephone utilities, computing resources are now delivered on-demand as a utility service.
- **Shifting financial models**: Moving from CapEx to OpEx allows companies to avoid massive upfront costs and the risk of under- or over-provisioning hardware.
- **Economies of scale**: Massive cloud providers (like AWS, Google, Azure) achieve extreme efficiency and cost savings because they operate at an immense scale, even designing custom hardware.
- **Focus on core business**: Companies should focus on their actual product or service (e.g., selling pizzas) rather than the "heavy lifting" of running and maintaining data centers.
- **Speed and Agility**: Cloud computing allows businesses to deploy resources globally within minutes and scale infinitely with demand.

# 5. Important examples from the lecture

- **Startups & Scaling**: A hypothetical startup that unexpectedly goes viral on TikTok. With cloud computing, they can instantly scale their infrastructure to handle the sudden influx of global users without buying physical servers.
- **Economies of scale**: The lecturer notes that giant cloud providers design their own hardware and CPUs specifically to extract maximum value, highlighting the scale advantage public clouds have over private data centers.

# 6. What the professor emphasized

- **Not managing Data Centers**: Highly emphasized that managing physical facilities, power, and cooling is not the core business of most companies and should be offloaded to cloud providers.
- **Global reach and scalability**: Emphasized the ability to expand a service globally (e.g., to Japan, America, Europe) almost instantly without setting up physical infrastructure in those regions.

# 9. 5-20 flashcards in Q/A format

**Q:** What is Cloud Computing?
**A:** The on-demand delivery of IT resources over the Internet with pay-as-you-go pricing.

**Q:** What is CapEx in IT?
**A:** Capital Expenditure: Spending money upfront on physical infrastructure (e.g., servers, data centers).

**Q:** What is OpEx in IT?
**A:** Operational Expenditure: Variable expenses where you pay only for the computing resources you consume.

**Q:** What does IaaS stand for and what does the user manage?
**A:** Infrastructure as a Service. The user manages the OS, middleware, and applications.

**Q:** What does PaaS stand for and what does the user manage?
**A:** Platform as a Service. The user focuses only on deploying applications and data; the provider manages the OS and infrastructure.

**Q:** What does SaaS stand for?
**A:** Software as a Service. It is a fully functional application managed entirely by the provider (e.g., Gmail).

**Q:** Name the three types of cloud deployment models.
**A:** Public cloud, Private cloud, and Hybrid cloud.

**Q:** What is the primary financial advantage of cloud computing?
**A:** Trading capital expense (CapEx) for variable expense (OpEx).

**Q:** How does cloud computing affect capacity planning?
**A:** It allows companies to stop guessing about capacity by scaling resources up or down dynamically based on demand.

**Q:** What is Virtualization?
**A:** Software that manipulates hardware to present a consistent interface, serving as the foundational layer for cloud computing.
