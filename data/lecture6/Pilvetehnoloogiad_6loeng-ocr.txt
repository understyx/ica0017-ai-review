SECURITY GROUPS (SG)
• You can specify allow rules, but not deny rules.
• When you first create a security group, it has no inbound rules. Therefore, no inbound
traffic is allowed until you add inbound rules to the security group.
• When you first create a security group, it has an outbound rule that allows all outbound
traffic from the resource. If your security group has no outbound rules, no outbound traffic
is allowed.
• When you associate multiple security groups with a resource, the rules from each security
group are aggregated to form a single set of rules that are used to determine whether to
allow access.
• When you add, update, or remove rules, your changes are automatically applied to all
resources associated with the security group.

SECURITY GROUPS (SG)
• Protocol: The protocol to allow. The most common protocols are 6 (TCP), 17 (UDP), and 1
(ICMP).
• Port range: For TCP, UDP, or a custom protocol, the range of ports to allow. You can specify
a single port number (for example, 22), or range of port numbers (for example, 7000-8000).
• ICMP type and code: For ICMP, the ICMP type and code. For example, use type 8 for ICMP
Echo Request or type 128 for ICMPv6 Echo Request.
• Source or destination: The source (inbound rules) or destination (outbound rules) for the
traffic to allow. Specify a single IPv4 or IPv6 address or range.

ELB – (ELASTIC LOAD BALANCER)

The load balancer serves as a single point of contact for all the instance.

ELB – Elastic Load Balancer*
ALB – Application Load Balancer
NLB – Network Load Balancer

ELB – ELASTIC LOAD BALANCER
§ Introduced in 2009
§ Think of it as a Nginx or HAProxy
§ Works on L4 (TCP) and L7 (HTTP)
§ Isn’t compatible with EKS containers running on Fargate
§ Can’t forward traffic on more than one port per instance
§ Doesn’t support forwarding to IP addresses

ALB / NLB
In 2016, AWS launched its Elastic Load Balancing version 2, which is made
up of two offers: Application Load Balancer (ALB) and Network Load
Balancer (NLB).
Both ALB and NLB can forward traffic to IP addresses, which allows them to
have targets outside the AWS Cloud (for example: on-premises servers or
instances hosted on another cloud provider).

APPLICATION LOAD BALANCER
§ Only works at layer 7 (HTTP)
§ Supports HTTP/2 and websockets
§ Can route requests to many ports on a single target
§ Can route requests to Lambda functions
§ Can be configured to return a fixed response or a redirection
§ Supports user authentication via OIDC, SAML, LDAP, Microsoft AD, also
Facebook and Google.

NETWORK LOAD BALANCER
§ Works at layer 4 only and can handle both TCP and UDP, as well as TCP
connections encrypted with TLS
§ Main feature - has a very high performance
§ Uses static IP addresses and can be assigned Elastic IPs—not possible
with ALB and ELB
§ Natively preserves the source IP address in TCP/UDP packets;
